Critical Bugs
#01
CRITICAL
패턴 인덱스 오류
— 30층 보스 첫 턴 즉사
combat.py
발견 경위
코드 리뷰 중 발견. 모든 몬스터가 패턴의 첫 번째 행동을 건너뛰고 있었음. 30층 보스(심연의 근원)는 패턴이 ["quick", "dark_slash", "aoe"]인데, 첫 턴에 dark_slash(2배 데미지)가 발동 → 파티 즉사 수준.
원인
turn이 1부터 시작하므로 turn % len(pattern)은 index 1부터 계산. pattern[0]은 게임 내내 두 번에 한 번만 등장.
▶ BEFORE (버그)
pattern[turn % len(pattern)]

# turn=1, len=3 →
# pattern[1] = dark_slash 💀
▶ AFTER (수정)
pattern[(turn - 1) % len(pattern)]

# turn=1, len=3 →
# pattern[0] = quick ✓
패턴 시작 index 0 보장 보스전 밸런스 정상화
#02
CRITICAL
aoe 실제 데미지와
인텐트 표시 불일치
combat.py
발견 경위
코드 리뷰 중 발견. 플레이어에게 보여지는 예고 데미지와 실제 데미지가 달랐음. 플레이어는 1.1배를 보고 대비하지만 실제론 0.7배 — 인텐트 시스템 자체가 무의미.
원인
fire_aoe와 aoe가 같은 if 블록에서 처리되어 두 행동 모두 0.7배 고정. aoe 인텐트 표시는 1.1배로 작성되어 있었음.
▶ BEFORE (버그)
if action in ("fire_aoe", "aoe"):
dmg = _calc_dmg(monster, c, 0.7)
# aoe도 0.7배 — 잘못됨
▶ AFTER (수정)
if action == "fire_aoe":
dmg = _calc_dmg(monster, c, 0.7)
if action == "aoe":
dmg = _calc_dmg(monster, c, 1.1)
인텐트 시스템 신뢰성 회복 aoe = fire_aoe보다 강한 광역공격으로 분리

원인 2. monster_first=True 타이밍 → 적이 먼저 공격한 뒤 플레이어가 기절 적용 → "기절 걸었는데 맞았다" 착각 (실제로는 기절 전에 공격)
▶ BEFORE
# 카드 설명
"기절 1턴"

# 실패 메시지 (눈에 안 띔)
[dim yellow]버텼습니다[/dim yellow]
▶ AFTER (수정)
# 카드 설명
"기절 1턴 (50% 확률)"

# 실패 메시지 (강조 + 딜레이)
[bold yellow]버텼습니다![/bold yellow]
time.sleep(0.5)
확률 정보 명시 실패 피드백 가시성 향상

Design Changes
D1
DESIGN
공용 카드 획득 경로
재설계
combat.py · world.py · assets.py
초안
전투 승리 보상 카드 3장 중 1/3 확률로 공용 카드(마나 회복, 집중, 긴급 교대, 해독제) 등장.
문제
공용 카드가 0 MP이고 강력한 범용 효과 → 전투 보상에서 자주 뜨면 밸런스 붕괴 가능. 클래스 전용 카드 획득 기회도 감소.
수정
전투 보상에서 완전 제거 → 특정 사건(이벤트)에서만 획득 가능하도록 재설계. 사건 4개 신규 추가.
떠돌이 약초상 → 해독제 (20G) 명상하는 수도승 → 집중 (무료) 비밀 통로 → 긴급 교대 (무료) 마력의 샘 → 마나 회복 (HP -10)
D2
DESIGN
퀵 스왑 덱 사이클링
— 의도된 기능으로 확정
player.py · combat.py
현상
스왑 시 draw_cards() 호출로 새 캐릭터의 손패가 리셋됨. 덱이 소진되면 버린패 셔플(리로드). 빠른 스왑으로 덱을 빠르게 순환 가능.
논의
버그인지 기능인지 확인. 의도치 않은 부산물이지만 전략적 플레이 요소(퀵 스왑 덱) 가능성 존재.
결론
사용자 요청으로 현재 동작 유지. 퀵 스왑 덱 빌딩 전략으로 인정.
의도된 전략 요소로 확정 코드 수정 없음
